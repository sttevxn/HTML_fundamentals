Estructura basica de un Formulario

La caracteristica mas imporante de los formularios es que nos permite interactuar con el usuario.

Casi todas las paginas web del mundo, exceptuando las informativas, poseen formularios.

Semantica de los formularios.

1- form: Es la etiqueta que engloba y contiene en su interior nustro formulario.

2- label: Sirve para escribir el nombre del campo a rellenar.

3- input: Es el campo en el cual el usuario interactua y rellena con informacion.

4- button: Es el elemento con el que se envia la informacion recibida.

Semantica

Se inciara el formulario abriendo una etiqueta form y en su interior se dará nombre al campo que se desee crear con la etiqueta label, luego de esto se debe insertar un correspondiente input, en el cual el usuario depositara la informacion y como paso final, se agrega un boton para enviar la informacion haciendo uso de la etiqueta button.

Asociacion de Input y Label.

Sabiendo ya la estructura basica de los formularios, nos adentramos en los atributos.

La etiqueta label posee el atributo for al cual se le indica un identificador (id) que emparejará al label con su correspondiente input.
La etiqueta input por su parte, debe tener el atributo id para asociar el label con el input. Desde ese momento, cuando hagamos click en el nombre del formulario, se activará automaticamente el campo a rellenar.

Tipos de Input

El elemento input posee el atributo type, que despliega una lista de distintos tipos de input segun la necesidad que amerite.

  • Button: Se comporta igual que un boton.
  • Color: Se utiliza para especificar un color.
  • Date: Se utiliza para introducir una fecha.
  • Email: Se usa para introducir un e-mail
  • File: Permite seleccionar un archivo del dispositi
  • Hidden: Campo oculto, no se mostrara el valor qie se ingrese.
  • Month: Para indicar un mes.
  • Number: Para ingresar valores numericos.
  • Password: Para indicar contraseñas.
  • Range: Para indicar un rango.
  • Reset: Reinicia el formulario.
  • Search: Para las barras de busqueda.
  • Submit: Envia el formulario, funciona con el type="button"
  • Tel: Para ingresar numeros telefonicos.
  • Text: Valor por defecto.
  • Time: Para indicar una hora.
  • Url: Para indicar una url.

Etiquetas Name y Value.

La funcion de estas etiquetas presentes en la etiqueta input tienen como funcion:

  1. Name: Nombre del tipo de informacion que se esta por ingresar
  2. Value: La informacion en sí.

Cuando se envia la informacion ingresada, el servidor que la reciba tendra la informacion de la categoria que se esta recibiendo y su respectivo valor para ser manipulado.

Uso de Name y Value en Inputs de Seleccion.

El atributo Name es obligatorio ya que permite "agrupar" a los inputs segun el nombre que se le atribuya.

De la misma manera, el atributo Value es obligatorio para que el navegador identifique cual es la opcion elegida por el usuario en el momento de enviar la informacion del formulario.

  • Input Radio
  • Input CheckBox
  • Input Select: Despliega una lista de opciones para eligir.

1- Input Radio

Permite que el usuario elija unicamente una opcion de las que haya disponibles.

2- Input CheckBox

Permite que el usuario pueda elegir más de una opcion.

3- Input Select

Funciona como una lista desplegable que le permite al usuario elegir una o mas opciones segun se desee.
A diferencia de los inputs vistos anteriormente, Slecet tiene su propia etiqueta dentro de la cual, se especifica con la etiqueta option cuales seran las opciones disponibles.

Input Select: Multiple Opcion

Para tener la funcion de elegir mas de una opcion, se usa el atributo multiple para que, presionando la tecla CRTL+CLICK, el usuario elija mas de una opcion.

Input Select: Grupos de Opcion

Cuando en ocaciones se tiene una lista de opciones muy larga, se pueden agrupar estas mismas en "categorias". Esto es posible mediante el uso de la etiqueta optgroup dentro de la etiqueta select, y haciendo uso de su respectivo atributo label, donde se definira el titulo del grupo armado con las opciones.

Busqueda con Filtro. Uso de Datalist

Cuando se desee que el usuario realice una busqueda en una lista de opciones, se se hace uso simultaneo del Input List y la etiqueta Datalist, la cual en su interior tendra las opciones enmarcadas en etiquetas option (similar al input select)

IMPORTANTE: datalist debe estar identificado medianteun id con la lista armada

Fieldset: Englobando Datos

El elemento Fieldset ayuda al usuario a guiarse en el momento en que se deban rellenar muchos campos de informacion en un formulario.

Semantica

Fieldset sera usada dentro de la etiqueta form y para darle nombre a la seccion en la que se encuentre el usuario, se hará uso del elemento legend.

Textarea: Area de texto

Textarea es el elemento que se usa con mayor frecuencia para rellenar espacios que seran usados para enviar mensajes, escribir textos largos y demas. Posee atributos que permiten modificar sus dimensiones pero esto es modificado normalmente con CSS.

Semantica

Se usara textarea dentro de la etiqueta form y será asociada a un corresponiente label si se desea.

Min - Max. Limites en inputs numericos

Min y Max son atributos que se usan en los inputs de tipo numericos y permiten marcar un rango entre los numero que se desean que esten disponibles para la eleccion del usuario.

Ejemplo

Minlenght - Maxlenght: Limites en inputs de texto

Minlenght y Maxlenght permiten que se determine un limite en el largo de los datos que se ingresen en los inputs de tipo texto. Usualmente se usan para numeros de documentos, contraseñas, nombres de usuario entre otros ejemplos.

Metodos GET y POST

Cuando se envian los datos ingresados en el formulario, inidcando el destino con el atributo action, se debe definir qué metodo sera usado para enviar dichos datos.

Mientras que GET envia informacion mediante la URL, POST lo hace en un segundo plano "ocultos" para el usuario, pero pueden ser visualizados mediante el inspector de elementos (F12) en el apartado Network; y es en este punto en el que el atributo name gana importancia ya que gracias a éste, el navegador reconoce a que pertenece cada dato recibido, ya sea nombre, edad, o cualquiera sea el caso.
